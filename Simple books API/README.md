# Simple Books API Testing Project

## Overview

This repository contains a Postman collection developed to test the functionality of the Simple Books API.

The project validates key API endpoints through automated assertions, status code verification, and response validation. It also demonstrates the use of dynamic variables to manage authentication tokens and order IDs generated during test execution.

## Features

* Automated API testing using Postman
* HTTP status code validation
* Response body verification
* Authentication testing
* Order management testing
* Dynamic variable handling
* Request chaining through collection variables

## Test Coverage

The collection includes tests for:

* API status verification
* Book catalog retrieval
* Individual book retrieval
* Authentication
* Order creation
* Order retrieval
* Order updates
* Order deletion

## Validation Performed

Automated assertions are included to verify:

* Expected HTTP status codes
* Successful API responses
* Response data accuracy
* Authentication functionality
* Order creation and retrieval
* Order update operations
* Order deletion operations

## Variable Management

The collection uses Postman collection variables to store and reuse dynamic data generated during execution, including:

* Authentication tokens
* Order IDs

This approach supports end-to-end workflow validation and reduces manual intervention during test execution.

## Security

Authentication tokens and other sensitive values have been removed from this repository before publication.

## How to Run

1. Import the collection into Postman.
2. Configure the required variables.
3. Execute the collection using the Collection Runner.
